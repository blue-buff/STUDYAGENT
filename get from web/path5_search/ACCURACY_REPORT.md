### 2.1 完整性评估

| 完整度 | 数量 | 说明 |

| 完整（题目+答案+解析） | 4 (24%) | 从掘金、Slideserve 等技术站点提取 |

## 三、已知问题列表

### 3.1 搜索质量问题

| # | 问题 | 影响 | 严重程度 |
|---|------|------|---------|
| 1 | DuckDuckGo 搜索中文内容结果不稳定 | 同一 query 不同时间结果差异大 | 中 |
| 2 | 搜索可能返回不相关网站（视频课、小说站） | 浪费时间抓取无关页面 | 中 |
| 3 | 搜索结果偏向页面标题匹配，而非内容匹配 | 返回大量目录页而非具体题目 | 高 |
| 4 | 搜索结果中大部分是 JS 渲染页面 | 无法用 requests 抓取内容 | 高 |

**缓解措施：**
- 使用题目文本模式搜索（"已知函数f(x)"）比主题搜索效果好得多
- JS 站点（zujuan, jyeoo）被自动跳过，需 Playwright 路径处理
- 已实现搜索摘要直接提取，绕过页面抓取

### 3.2 页面抓取问题

| # | 问题 | 影响 | 严重程度 |
|---|------|------|---------|
| 5 | 中国文档分享站点（道客巴巴、豆丁网、百度文库）全需登录 | 抓取内容为空或验证页面 | 高 |
| 6 | 知乎需 Cookie 才能正常访问 | requests 直接访问返回空 | 中 |

### 3.3 LLM 提取问题

| # | 问题 | 影响 | 严重程度 |
|---|------|------|---------|
| 9 | DeepSeek V4 Pro 使用 thinking mode，返回 ThinkingBlock+TextBlock | 需特殊处理响应结构 | 中（已修复） |
| 10 | LLM 无法区分相似题目（同一页面多道离子方程式题） | 可能漏掉有效题目 | 低 |

| 12 | 答案字段可能为 None（应统一为空字符串） | JSON 解析报错 | 低（已修复） |

### 3.4 幻觉风险

| # | 风险 | 当前缓解 |
|---|------|---------|
| 13 | LLM 编造不存在的题目 | 要求 question_found: false 机制 + 强制标注来源 URL |
| 14 | 从片段拼凑出"完整"但实际不存在的题目 | 高置信度要求 + extraction_notes 说明信息完整性 |
| 15 | 猜测答案 | prompt 明确要求"不要猜测答案" |

**观察：** 16/17 道题来自可追溯的具体 URL，置信度均 ≥ 0.9。未发现明显幻觉案例。所有 extraction_notes 都诚实地说明了信息缺失情况。

## 五、与搜索 API 的协同分析

| 搜索后端 | 中文质量 | 可用性 | 推荐 |
|----------|---------|--------|------|
| DuckDuckGo | ★★★ | 完全免费，无 API Key | 开发/测试推荐 |
| Brave Search | ★★★★ | 免费 2000次/月 | 小规模生产推荐 |
| Bing Web Search | ★★★★★ | 免费 1000次/月 | 生产环境推荐 |

当前使用 DuckDuckGo 作为默认后端，测试中表现尚可。升级到 Bing Search API 预期可显著提升中文搜索结果质量。

## 六、改进建议

1. **【高优】使用 Bing Search API**：中文搜索质量最高，可显著减少不相关结果
2. **【高优】缓存搜索-提取结果**：同一 query 的搜索结果应缓存，避免重复搜索和 API 调用
3. **【中优】添加 Playwright 分支**：对 zujuan.xkw.com 等 JS 站点使用 Playwright 渲染
4. **【中优】支持批量提取**：从同一页面提取多道题目（当前每页仅提取一道）
5. **【低优】数学公式规范化**：将文本中的 LaTeX/Unicode 数学符号统一为 LaTeX 格式
6. **【低优】答案校验**：用第二次 LLM 调用验证提取的答案是否正确（基于题目内容）
