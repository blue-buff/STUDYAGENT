## 二、详细评估

### 1. DuckDuckGo (duckduckgo_search Python库)

**优点：**
- 完全免费，无 API Key 需求
- 无速率限制公告（实测需适当控制频率）
- 支持中文搜索，结果质量尚可
- Python 库封装良好，开箱即用
- 隐私友好，无用户追踪

**缺点：**
- 非官方 API，未来可能变更
- 搜索结果略少于 Google/Bing
- 无高级筛选参数（时间范围、地区等不如商业 API 丰富）
- 偶尔会触发速率限制（需重试机制）

**实际测试：**
```
Q: "高中数学 导数单调性 选择题 中等难度 题目 解析 答案"
Results: 10-20条，相关性良好
平均响应时间: 1-3秒
```

**推荐场景：** 开发阶段 + 低预算项目 + 一般精度要求

---

### 2. Brave Search API

**优点：**
- 独立搜索引擎索引，不受 Google/Bing 约束
- 免费 tier 每月 2,000 次查询，个人项目足够
- 隐私友好
- API 响应快，结构化好
- 支持地区和语言参数

**缺点：**
- 索引规模小于 Google/Bing
- 中文内容覆盖面可能不如 Bing
- 需要注册账号获取 API Key

**申请地址：** https://brave.com/search/api/

**推荐场景：** 有一定搜索量需求的正式项目

---

### 3. Bing Web Search API (Azure Marketplace)

**优点：**
- **中文搜索质量最高**（Bing 对中文内容索引好）
- 结果丰富：包含网页、图片、视频等多种类型
- 企业级可靠性和 SLA
- 支持按地区、语言、新鲜度等高级筛选

**缺点：**
- 免费 tier 仅 1,000次/月（Azure 市场）
- 需要 Azure 账号和信用卡
- 定价较复杂

**申请地址：** https://portal.azure.com → 创建 Bing Search 资源 → 选择免费 tier

**推荐场景：** 生产环境 / 对中文内容质量要求高

---

### 4. SerpAPI

**优点：**
- 封装了 Google、Bing 等多个搜索引擎
- 自动处理 CAPTCHA 和 IP 轮换
- 结构化 JSON 响应

**缺点：**
- 最贵：最低 $50/月
- 免费 100 次仅限试用
- 对于本项目过于重型

**推荐场景：** 大规模商业项目 / 需要 Google 搜索

---

### 5. Google Custom Search API

**优点：**
- Google 搜索质量高

**缺点：**
- 每天仅 100 次免费，需绑定信用卡
- 只能搜索指定站点（CSE），不能搜全网络
- 全网络搜索需要额外的 API 额度

**推荐场景：** 针对特定站点（如只搜 jyeoo.com）

---

## 三、本项目选型结论

### 主方案：DuckDuckGo (当前使用)

- **理由：** 零成本、零配置、中文支持可用、开发效率最高
- **风险：** 非官方 API 稳定性
- **缓解：** 代码中已支持 Brave/Bing 作为备选切换

### 升级路径

```
开发阶段  → DuckDuckGo (免费)
验证通过  → Brave Search API (免费 tier, 2000次/月)
生产部署  → Bing Web Search API (按需付费, 中文最优)
```

### 切换方法

```bash
# 切换到 Brave
export SEARCH_BACKEND=brave
export BRAVE_API_KEY=BSA...

# 切换到 Bing
export SEARCH_BACKEND=bing
export BING_API_KEY=...
```
