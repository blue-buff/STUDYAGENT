## URL 规则分析

### 数学 (已验证)
```
https://zujuan.xkw.com/{gzsx|czsx}/zsd{id}/qt{题型码}d{难度}y{年份}o{排序}p{页码}/
```
- 题型码: 高中(2701-2704), 初中(1101-1104)
- 排序: o2=最新, o1=最热, o0=综合

### 物理 (初步验证)
```
https://zujuan.xkw.com/gzwl/zsd{id}/x4o2/
```
- `x4` 前缀含义可能是"新题"筛选
- 知识点 ID: zsd41934 (综合库) 等
- 题目结构相同: `div.tk-quest-item.quesroot`

### 化学 (待深入验证)
- 预计 URL 格式类似物理
- 题目结构预计相同

## 知识点树获取方式

当前项目使用预生成的文本文件 (KNOWLEDGE_TREE_HIGH.txt) 构建 SQLite 数据库。

### 扩展方案 A: 爬取各学科知识点树
在组卷网各学科页面，知识点树通过前端 API 加载：
1. 访问 `https://zujuan.xkw.com/{prefix}/`  
2. 页面 JS 会请求知识点树 API
3. 可从页面 DOM 或网络请求中提取树结构
4. 保存为 KNOWLEDGE_TREE_{SUBJECT}.txt 统一格式

### 扩展方案 B: 手动整理
参考教材目录结构，手动构建知识点树。
适合知识点相对固定的学科（如数学、物理）。

建议：方案 A 为主，方案 B 为辅。

## 题目结构一致性

所有学科使用相同的 HTML 结构：
```
div.tk-quest-item.quesroot           # 题目容器
  div.ques-additional                 # 来源、题型、难度
  div.wrapper.quesdiv                 # 题目内容 + 答案
    div.exam-item__cnt                # 题目内容区
    div.exam-item__opt                # 答案区
      div.item.answer img             # 答案图片
```

这意味着现有抓取代码 (`scrape.py`) 理论上可以直接用于物理、化学等学科，只需：
1. 修改 URL 前缀和排序参数
2. 提供对应学科的知识点 ID

## 题型码差异

不同学科的题型码可能不同，需要逐学科验证：

| 学科 | 已知题型 | 题型码格式 |
|------|----------|-----------|
| 数学 | 单选/多选/填空/解答 | qt2701~2704 |
| 物理 | 待验证 | 可能不同 |
| 化学 | 待验证 | 可能不同 |

## 扩展开发步骤

1. **获取知识点树** — 为物理、化学爬取或手动创建 KNOWLEDGE_TREE 文件
2. **验证题型码** — 在各学科页面抓取题型筛选链接，确定 qt 编码
3. **验证排序参数** — 确认 x4/o2/o1/o0 等排序参数在各学科的对应关系
4. **修改 URL Builder** — 支持不同学科的排序和题型参数
5. **测试抓取** — 用现有 scrape.py 框架抓取物理题验证

## 风险评估

- **低风险**: 题目 HTML 结构一致，抓取逻辑可复用
- **中风险**: 题型码和排序参数可能有差异，需逐学科验证
- **低风险**: 登录态跨学科共享（已验证 Cookie 涵盖 .xkw.com）
