# 实现路径分析 — 题目检索 Agent

> 目标：从主流题目网站按需求检索、筛选题目，为高中生推荐练习
> 约束：先保证工具能调通，暂不实现 Agent 层

---

## 路径总览

```
Path 1: 组卷网 Playwright 自动化    ← 浏览器截图，最成熟
Path 2: 组卷网 HTTP API 逆向        ← 轻量快速，抓结构化数据
Path 3: 菁优网 Python 爬虫          ← 多学科覆盖（物化英）
Path 4: 公开题库 + 本地检索         ← 无爬取风险，但题库可能不全
Path 5: 搜索引擎聚合 + LLM 解析     ← 覆盖面最广，但质量不可控
Path 6: 学科网 试卷级资源           ← 整卷下载，适合出卷场景
```

---

## Path 1：组卷网 Playwright 浏览器自动化

**技术栈**：TypeScript + Playwright + SQLite
**复用项目**：`malyjacob/zujuan`（已验证可编译运行）

### 现状
- ✅ CLI 完整可用（登录、知识点浏览、抓取、导出）
- ✅ 知识点树完整（18个顶级领域，3级深度，含全部高中数学考点）
- ✅ 支持按题型/难度/年份/排序筛选
- ✅ 题目截图 + 答案下载 + 视觉 OCR
- ✅ MIT 许可证

### 需要做的
1. **验证端到端流程**：扫码登录 → 实际抓取题目 → 导出 HTML/Markdown
2. **封装为可调用工具**：将 CLI 命令封装为 Python/HTTP API，供后续 Agent 调用
3. **扩展学科**：补充物理、化学等学科的知识点树和 URL 规则
4. **优化抓取速度**：并发控制、缓存已抓取题目

### 优劣势
| 优势 | 劣势 |
|------|------|
| 代码成熟，改动最小 | 需维护浏览器实例，资源消耗大 |
| 截图精确，公式渲染好 | 微信扫码登录需人工介入 |
| 能处理任意 JS 动态渲染 | 抓取速度慢（每题需等待渲染） |

## Path 2：组卷网 HTTP API 逆向

**技术栈**：Python + requests + BeautifulSoup
**复用项目**：`kaibush/tk-exam-paper`（22⭐，2026-05-04 更新）

### 现状
- 有 QR 码登录流程代码
- 有基础的 HTTP 请求 + Cookie 管理
- 有 Flask + PyQt5 GUI 界面
- 项目结构较简陋（11 次提交，无详细文档）

### 需要做的
1. **逆向组卷网 API**：抓包分析题型列表/题目详情接口（XHR/Fetch 请求）
2. **解析 token/签名机制**：找到请求参数中的加密字段生成逻辑
3. **构建 Python 版知识点树**：将 zujuan 项目的知识点树移植为 Python 可用格式
4. **写筛选 + 搜索脚本**：按知识点/题型/难度/年份搜索，返回结构化 JSON
5. **Cookie 持久化 + 自动刷新**：延长登录态有效期

### 优劣势
| 优势 | 劣势 |
|------|------|
| 轻量快速，无浏览器开销 | API 逆向工作量大，不确定性高 |
| 直接获取结构化数据（非截图） | 网站改版即失效 |
| 易于部署到服务器 | token 加密可能很复杂 |
| 可批量高速抓取 | 每个学科的 API 规则可能不同 |

## Path 3：菁优网 (jyeoo.com) Python 爬虫

**技术栈**：Python + Selenium/Playwright + MySQL
**复用项目**：`pengwow/jyeoo-crawler-gui`（30⭐）或 `pengwow/web-crawler`（14⭐）

### 现状
- GUI 版已归档（2019年），但 Python 版 2022 年有更新
- 支持按章节/课题 ID 批量爬取
- 支持详情页爬取（含知识点标签）
- 菁优网题库覆盖多学科：数学、物理、化学、英语等

### 需要做的
1. **更新爬虫代码**：PhantomJS 已废弃，需迁移到 Playwright 或 Selenium 新版
2. **适配当前网站结构**：jyeoo.com 2026 年的页面结构与 2019 年差异大
3. **实现登录流程**：QQ 登录 → Cookie 持久化
4. **构建学科/章节索引**：各学科的知识点树结构
5. **统一数据输出格式**：与 Path 1/2 输出统一的 JSON Schema

### 优劣势
| 优势 | 劣势 |
|------|------|
| 多学科覆盖（物化英等） | 代码老旧，需大量更新 |
| 题库量可能更大 | 反爬更严格（QQ登录） |
| 有知识点标签体系 | 开发周期较长 |
| 弥补组卷网仅数学的不足 | 项目已停更多年 |

## Path 4：公开题库数据集 + 本地检索

**技术栈**：Python + SQLite/Elasticsearch + 向量数据库
**思路**：寻找已有的公开题库数据集，建立本地检索引擎

### 可能的数据来源
- Hugging Face 教育数据集（中文数学题）
- 历年高考真题 PDF/文本（公开资源）
- GitHub 上的题库汇总仓库
- 教育类开放数据平台

### 需要做的
1. **搜集公开题库**：搜索 HuggingFace、GitHub、开放数据平台的题库数据
2. **解析 + 结构化**：将各种格式（PDF/图片/文本）转为统一 JSON
3. **构建检索引擎**：基于知识点 + 难度 + 题型的多条件筛选
4. **语义搜索**：可选，用 embedding 实现"类似题目"推荐

### 优劣势
| 优势 | 劣势 |
|------|------|
| 无法律/反爬风险 | 题库可能不全或过时 |
| 检索速度极快 | 数据质量参差不齐 |
| 可离线使用 | 需大量数据清洗工作 |
| 适合构建长期资产 | 公开高中题库资源稀少 |

## Path 5：搜索引擎聚合 + LLM 解析

**技术栈**：Python + Search API (Bing/Google) + LLM
**思路**：用搜索引擎搜索目标题目，LLM 解析结果并结构化

### 实现思路
1. 用户需求 → LLM 生成搜索 query（如 "高中数学 导数 单调性 选择题 中等难度"）
2. 调用搜索 API 获取结果页面
3. 抓取搜索结果中的题目页面
4. LLM 从抓取内容中提取题目、答案、解析，化为结构化数据

### 优劣势
| 优势 | 劣势 |
|------|------|
| 覆盖面最广，不拘泥于特定网站 | 题目质量不可控 |
| 无需维护登录态 | 依赖搜索引擎 API（有费用） |
| 天然支持所有学科 | LLM 提取准确性不稳定 |
| 开发周期短 | 可能有版权问题 |

## Path 6：学科网 (xkw.com) 试卷级资源

**技术栈**：Tampermonkey JS / Playwright
**复用项目**：`bzyzh/xkw-zujuan-script`（34⭐） + `Frank-678/zxxk-zujuan-paper-downloader`（6⭐）

### 现状
- 油猴脚本成熟，可导出/打印试卷
- 但需要 PLUS 会员才能访问优质试卷
- 学科网是组卷网的母站，资源最全（全学科全学段）

### 需要做的
1. **分析会员权限边界**：确认哪些功能免费可用
2. **尝试 Playwright 替代油猴脚本**：自动化执行脚本逻辑
3. **试卷→单题拆分**：将下载的试卷拆分为独立题目并标注属性
4. **扩展学科支持**：利用学科网的全学科覆盖

### 优劣势
| 优势 | 劣势 |
|------|------|
| 全学科全学段覆盖 | 核心功能需 PLUS 会员 |
| 试卷级资源质量高 | 油猴脚本不适合 Agent 调用 |
| 与 Path 1/2 同体系互补 | 学科网反爬可能更强 |

## 建议分配策略

如果同时指派多个 AI 实现，建议按 **依赖关系最小化** 原则分配：

```
组 A（优先）：Path 1 + Path 2
  → 同一数据源（组卷网），两种技术路线互为备份
  → 可以共享知识点树、URL 规则等资源
  → 一个能跑通就算成功

组 B（扩展）：Path 3
  → 不同数据源（菁优网），解耦独立
  → 打开多学科覆盖的可能性

组 C（兜底）：Path 5 或 Path 4
  → 完全不依赖特定网站
  → 作为长期稳定方案
  → 可以先做可行性验证（搜集公开题库资源）
```

## 统一数据接口（建议所有 Path 输出格式一致）

```json
{
  "id": "唯一标识",
  "source": "zujuan / jyeoo / public / search",
  "subject": "数学",
  "grade": "高中",
  "knowledge_points": ["导数", "单调性"],
  "question_type": "单选题",
  "difficulty": 3,
  "year": 2024,
  "question_text": "已知函数 f(x) = ...",
  "question_images": [],
  "answer_text": "故选 A",
  "answer_images": [],
  "analysis": "本题考查导数在研究函数单调性中的应用...",
  "source_url": "https://...",
  "metadata": {}
}
```

所有 Path 的最终交付物都是：**给定筛选条件 → 返回上述格式的题目列表**。
